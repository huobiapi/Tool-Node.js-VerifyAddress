# Security Policy

The following versions of proof-of-address are currently being supported with security updates:

| Version | Supported          |
| ------- | ------------------ |
| 1.6.x   | :white_check_mark: |


We recommend always using the latest version for the best security.

## Reporting a Vulnerability

If you discover a security vulnerability, please **DO NOT** report it in a public issue.

### How to Report

Please report security issues through one of the following methods:

1. **GitHub Security Advisory**: Create a private report on the project's [Security Advisory](https://github.com/huobiapi/Tool-Node.js-VerifyAddress/security/advisories) page
2. **Email**: Send an email to the project maintainers (contact information can be found in GitHub profiles)

### What to Include in Your Report

- Detailed description of the vulnerability
- Steps to reproduce the issue
- Affected versions
- Potential impact and scope
- Suggested fix (if available)

### Response Timeline

- **Initial Response**: We will acknowledge receipt of your report within 48 hours
- **Progress Updates**: We will provide investigation progress updates every 7 days
- **Fix Timeline**: We will release a patch as soon as possible based on the severity of the vulnerability

### Vulnerability Handling Process

1.  **Accepted**: We will work with you to fix the issue and credit you after the patch is released
2.  **Declined**: If our investigation determines it is not a security issue, we will explain why

## Security Best Practices

When using this tool, please note:

- Always use the latest version
- Never expose private keys or mnemonic phrases in public environments
- Test thoroughly in a test environment before using in production
- Regularly update dependencies to receive security patches

## Acknowledgments

Thank you to all security researchers and users who responsibly disclose security issues.
